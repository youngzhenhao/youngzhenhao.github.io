基本

函数可见性说明符(Function Visibility Specifiers)

public

visible externally and internally (creates a getter function for storage/state variables)
外部和内部都可见(为存储/状态变量创建getter函数)

private

only visible in the current contract
仅在当前合约中可见

external

only visible externally (only for functions) - i.e. can only be message-called (via this.func)
仅在外部可见(仅对函数可见)-即只能被message调用(通过this.func)

internal

only visible internally
仅在内部可见

修饰符(Modifiers)

pure

for functions: Disallows modification or access of state.
对于函数:禁止修改或访问状态。

view

for functions: Disallows modification of state.
对于函数:不允许修改状态。

payable

for functions: Allows them to receive Ether together with a call.
对于函数:允许它们在调用时接收Ether。

constant

for state variables: Disallows assignment (except initialisation), does not occupy storage slot.
对于状态变量:不允许赋值(初始化除外)，不占用存储槽。

immutable

for state variables: Allows exactly one assignment at construction time and is constant afterwards. Is stored in code.
对于状态变量:在构造时只允许一个赋值，然后是常量。以代码形式存储。

anonymous

for events: Does not store event signature as topic.
对于事件:不将事件签名存储为topic。

indexed

for event parameters: Stores the parameter as topic.
对于事件参数:将参数存储为topic。

virtual

for functions and modifiers: Allows the function’s or modifier’s behaviour to be changed in derived contracts.
对于函数和修饰符:允许在派生合约中改变函数或修饰符的行为。

override

States that this function, modifier or public state variable changes the behaviour of a function or modifier in a base contract.
声明此函数、修饰符或公共状态变量改变基合约中函数或修饰符的行为。

保留关键字(Reserved Keywords)

These keywords are reserved in Solidity. They might become part of the syntax in the future:
这些关键字在solidity中保留。它们将来可能会成为语法的一部分:

after, alias, apply, auto, case, copyof, default, define, final, immutable, implements, in, inline, let, macro, match, mutable, null, of, partial, promise, reference, relocatable, sealed, sizeof, static, supports, switch, typedef, typeof, unchecked.

合约结构(Structure of a Contract)

状态变量(State Variables)

State variables are variables whose values are permanently stored in contract storage.
状态变量是永久地存储在合约存储中的值。

函数(Functions)

Functions are the executable units of code. Functions are usually defined inside a contract, but they can also be defined outside of contracts.
函数是代码的可执行单元。函数通常定义在合约内部，但它们也可以定义在合约之外。

Function Calls can happen internally or externally and have different levels of visibility towards other contracts. Functions accept parameters and return variables to pass parameters and values between them.
函数调用可以在内部或外部发生，并且对其他合约具有不同级别的可见性。函数接受参数并返回变量，以便在它们之间传递参数和值。

事件(Events)

Events are convenience interfaces with the EVM logging facilities.
事件是EVM日志记录工具的方便接口。

错误(Errors)

Errors allow you to define descriptive names and data for failure situations. Errors can be used in revert statements. In comparison to string descriptions, errors are much cheaper and allow you to encode additional data. You can use NatSpec to describe the error to the user.
错误允许您为失败情况定义描述性名称和数据。在revert语句中可以使用错误。与字符串描述相比，错误要cheaper得多，并且允许您编码额外的数据。您可以使用NatSpec向用户描述错误。

结构类型(Struct Types)

Structs are custom defined types that can group several variables
结构是可以将多个变量分组的自定义类型

枚举类型(Enum Types)

Enums can be used to create custom types with a finite set of ‘constant values’
枚举可以用来创建具有有限“常量”集的自定义类型

类型(Types)

Solidity is a statically typed language, which means that the type of each variable (state and local) needs to be specified. Solidity provides several elementary types which can be combined to form complex types.
可靠性是一种静态类型语言，这意味着需要指定每个变量(状态和本地)的类型。可靠性提供了几种基本类型，这些类型可以组合成复杂类型。

The concept of “undefined” or “null” values does not exist in Solidity, but newly declared variables always have a default value dependent on its type. To handle any unexpected values, you should use the revert function to revert the whole transaction, or return a tuple with a second bool value denoting success.
“undefined”或“null”值的概念在solididity中不存在，但是新声明的变量总是有一个依赖于其类型的默认值。要处理任何意外值，您应该使用revert函数来恢复整个事务，或者返回一个元组，该元组的第二个bool值表示成功。

值类型(Value Types)

The following types are also called value types because variables of these types will always be passed by value, i.e. they are always copied when they are used as function arguments or in assignments.
以下类型也被称为值类型，因为这些类型的变量总是通过值传递，也就是说，当它们被用作函数参数或赋值时，它们总是被复制。

布尔值(Booleans)

bool: The possible values are constants true and false.
取值为常量true或false。

Operators:
运算符
! (logical negation)
逻辑非
&& (logical conjunction, “and”)
逻辑与
|| (logical disjunction, “or”)
逻辑或
== (equality)
等于
!= (inequality)
不等于

The operators || and && apply the common short-circuiting rules. This means that in the expression f(x) || g(y), if f(x) evaluates to true, g(y) will not be evaluated even if it may have side-effects.
运算符||和&&应用常见的短路规则。这意味着在表达式f(x) || g(y)中，如果f(x)的值为true, g(y)将不会被求值，即使它可能有副作用。

  • 逻辑与中，表达式1为真才会进行表达式2的运算
  • 逻辑或中，表达式1为假才会进行表达式2的运算

整数(Integers)

int / uint: Signed and unsigned integers of various sizes. Keywords uint8 to uint256 in steps of 8 (unsigned of 8 up to 256 bits) and int8 to int256. uint and int are aliases for uint256 and int256, respectively.

int / uint:各种大小的有符号和无符号整数。关键字uint8 to uint256 in steps of 8 (unsigned of 8 up to 256 bits) and int8 to int256。Uint和int分别是uint256和int256的别名。

Operators:
运算符
Comparisons: <=, <, ==, !=, >=, > (evaluate to bool)
比较

Bit operators: &, |, ^ (bitwise exclusive or), ~ (bitwise negation)
位运算符:按位与，按位或，按位异或，按位取反

Shift operators: << (left shift), >> (right shift)
移位运算符:<<(左移)，>>(右移)

Arithmetic operators: +, -, unary - (only for signed integers), *, /, % (modulo), ** (exponentiation)

算术运算符:+，-，一元-(只适用于有符号的整数)，*，/，%(取模)，**(取幂)

For an integer type X, you can use type(X).min and type(X).max to access the minimum and maximum value representable by the type.
对于整数类型X，你可以用type(X).min和type(X).max访问该类型可表示的最小值和最大值。

比较(Comparisons)

The value of a comparison is the one obtained by comparing the integer value.
比较的值是通过比较整数值得到的值。

位运算(Bit operations)

Bit operations are performed on the two’s complement representation of the number. This means that, for example ~int256(0) == int256(-1).
位运算是对两个数的补码表示进行的。这意味着，例如~int256(0) == int256(-1)。

  • 对0的补码取反为-1的补码
移位(Shifts)

The result of a shift operation has the type of the left operand, truncating the result to match the type. The right operand must be of unsigned type, trying to shift by a signed type will produce a compilation error.
移位操作的结果具有左操作数的类型，将截断结果以匹配该类型。右操作数必须为无符号类型，尝试按有符号类型移动将产生编译错误。

x << y is equivalent to the mathematical expression x * 2**y.
X << y等价于数学表达式X * 2**y。
x >> y is equivalent to the mathematical expression x / 2**y, rounded towards negative infinity.
X >> y等价于数学表达式X / 2**y，向负无穷四舍五入。

Before version 0.5.0 a right shift x >> y for negative x was equivalent to the mathematical expression x / 2**y rounded towards zero, i.e., right shifts used rounding up (towards zero) instead of rounding down (towards negative infinity).
在0.5.0版本之前，对于- x，右移x >> y等同于数学表达式x / 2**y向零四舍五入，即右移使用的是向上舍入(向零)而不是向下舍入(向负无穷)。

Overflow checks are never performed for shift operations as they are done for arithmetic operations. Instead, the result is always truncated.
移位操作不会像算术操作那样执行溢出检查。相反，结果总是被截断。

加法、减法和乘法(Addition, Subtraction and Multiplication)

The expression -x is equivalent to (T(0) - x) where T is the type of x. It can only be applied to signed types. The value of -x can be positive if x is negative. There is another caveat also resulting from two’s complement representation:
表达式-x等价于(T(0) -x)，其中T是x的类型。它只能应用于带符号的类型。如果x是负的，-x的值可以是正的。还有另一个由补码表示引起的警告:
If you have int x = type(int).min;, then -x does not fit the positive range. This means that unchecked { assert(-x == x); } works, and the expression -x when used in checked mode will result in a failing assertion.
如果有int x = type(int).min;，则-x不符合正范围。这意味着未检查{assert(-x == x);}工作，表达式-x在选中模式下使用将导致断言失败。

除法(Division)

Since the type of the result of an operation is always the type of one of the operands, division on integers always results in an integer. In Solidity, division rounds towards zero. This means that int256(-5) / int256(2) == int256(-2).

由于操作结果的类型始终是其中一个操作数的类型，所以对整数进行除法的结果总是一个整数。在Solidity中，除法趋向于零。即int256(-5) / int256(2) == int256(-2)。

The expression type(int).min / (-1) is the only case where division causes an overflow. In checked arithmetic mode, this will cause a failing assertion, while in wrapping mode, the value will be type(int).min.

表达式的类型(int)。Min /(-1)是除法导致溢出的唯一情况。在选中的算术模式下，这将导致断言失败，而在包装模式下，值将是type(int).min。

模(Modulo)

int256(5) % int256(2) == int256(1)

int256(5) % int256(-2) == int256(1)

int256(-5) % int256(2) == int256(-1)

int256(-5) % int256(-2) == int256(-1)

求幂(Exponentiation)

Exponentiation is only available for unsigned types in the exponent. The resulting type of an exponentiation is always equal to the type of the base. Please take care that it is large enough to hold the result and prepare for potential assertion failures or wrapping behaviour.
指数运算只对指数中的无符号类型可用。求幂的结果类型总是等于底数的类型。请注意，它要足够大以容纳结果，并为潜在的断言失败或包装行为做好准备。

In mode, exponentiation only uses the comparatively cheap exp opcode for small bases. For the cases of x**3, the expression x*x*x might be cheaper. In any case, gas cost tests and the use of the optimizer are advisable.
在checked模式下，对于较小的基数，求幂只使用相对便宜的exp操作码。对于x**3，表达式x*x*x可能更cheaper。在任何情况下，gas花费测试和使用优化器是明智的。

Note that 0**0 is defined by the EVM as 1.
EVM将0**0定义为1。

定点数(Fixed Point Numbers)

Warning

Fixed point numbers are not fully supported by Solidity yet. They can be declared, but cannot be assigned to or from.
固定数还没有被solidity完全支持。它们可以被声明，但不能被赋值给或从。

fixed / ufixed: Signed and unsigned fixed point number of various sizes. Keywords ufixedMxN and fixedMxN, where M represents the number of bits taken by the type and N represents how many decimal points are available. M must be divisible by 8 and goes from 8 to 256 bits. N must be between 0 and 80, inclusive. ufixed and fixed are aliases for ufixed128x18 and fixed128x18, respectively.
fixed / ufixed:各种大小的有符号和无符号定点数。关键字ufixedMxN和fixedMxN，其中M表示该类型的位数，N表示小数点个数。M必须能被8整除，从8到256位。N必须在0到80之间，包括。Ufixed和fixed分别是ufixed128x18和fixed128x18的别名。

Operators:
运算符
Comparisons: <=, <, ==, !=, >=, > (evaluate to bool)
比较
Arithmetic operators: +, -, unary -, *, /, % (modulo)
算术运算符

地址(Address)

address: Holds a 20 byte value (size of an Ethereum address).
保存一个20字节的值(以太坊地址的大小)。

address payable: Same as address, but with the additional members transfer and send.
与地址相同，但随附加成员转帐和发送。

The idea behind this distinction is that address payable is an address you can send Ether to, while you are not supposed to send Ether to a plain address, for example because it might be a smart contract that was not built to accept Ether.

这种区别背后的想法是，address payable是一个你可以发送Ether到的地址，而你不应该发送Ether到一个普通地址，例如，因为它可能是一个智能合同，不接受Ether。
